    Description

    Demonstrate hawk:AIs ability to avoid false positives on the same already validated scenario (=contextual replay). The scenario is executed in two steps:

    1. Initial transaction generates an Open Case in Case Manager as the account holder name “Wolfgang Berner” fuzzy matches 3 known Politically Exposed Person (PEP) and one Special Interest Person (SIP)
    2. Go into the case manager and set all matches to “No Match” and close the case
    3. Send the case again and it will appear in Auto-Close based on the previous decision made by the operator

    If the scenario is executed as is you omit the first step as the step 1 is already done. To explicitly test scenario 1 use a new random IBAN for Wolfgang Berner (http://randomiban.com/?country=Germany) as the contextual replay is done on the name/IBAN combination.

    Expected Result

    1. After setting the matches to “No Match” previously, case appears in “Auto-Close Cases” in Case Manager
